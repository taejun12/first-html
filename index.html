06/26/23 - web13-18

title - 파일의 제목을 나타냄
meta charset - 우리가 작성한 파일은 오른쪽 하단을 보면 UTF-8로 저장됨을 알 수 있음 -> 한글을 썼을 때 글씨가 깨지면 meta charset="UTF-8"을 입력 -> 글씨가 안 깨짐
head는 파일의 제목을 나타내는 tag -> title, meta charset과 같은 tag들은 head라는 tag로 묶어야함
본문은 body라는 tag로 묶음
body/head tag를 감싸는 태그가 존재 - html -> 이 문서가 html이라는 것을 나타내기 위해 html tag 위에는 !doctype html을 추가
a - 링크를 거는 tag -> html specification이라고 치면 W3C라는 사이트가 나옴 -> 이 사이트의 주소를 복사한 다음, 내 html 파일에서 링크를 걸고 싶은 문장에 a href="주소"와 같은 식으로 작성하면 해당 사이트로 가는 링크 생성
a href에서 a는 anchor, h는 hypertext, ref는 reference 의미
링크가 새 탭으로 열리게 하고 싶으면 target="_blank" 입력
클릭하기 전에 링크가 무엇인지 알려주고 싶으면 title="툴팁으로 알려주고 싶은 내용"입력하면 툴팁 생성

06/28/23 - web19

web server - 대표적인 것이 live server -> 실시간으로 웹이 갱신됨
http://127.0.0.1:5500/index.html를 해부해보면
http - hyper text transfer Protocol
127.0.0.1 - internet protocol address -> 내 컴퓨터를 나타내는 주소 - 개발할 때 많이 사용하는 ip 주소
5500 - port - 여러개의 서버가 존재할 때 각각의 서버가 port 번호를 가짐

- 제대로 작동하는지 확인하기 위해 2번의 테스트 필요
- 2개는 4번의 테스트 필요
- 3개는 8번의 테스트 필요
CSS - 웹사이트를 예쁘게 하기 위해 필요
javascript - 사용자와 상호작용하는 웹사이트를 만들기 위해 필요
JSP/PHP/Node.js - back end(하나의 파일을 고치면 여러 웹사이트들이 수정되는 기술)을 사용하기 위해 필요 셋 중에 하나 배우면 됨
동영상을 넣기 위해 iframe이라는 코드 사용 -
disqus/livere - 웹페이지에 댓글창을 만들기 위해 사용
